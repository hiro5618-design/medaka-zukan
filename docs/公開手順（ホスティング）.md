# メダカ図鑑をPC・スマホで見る（静的ホスティング手順）

図鑑をネット上に置いて、**URL 1つでPCもスマホも**見られるようにする手順です。
図鑑はデータをJSに埋め込んだ静的サイトなので、**サーバー不要**でそのまま置けます。

> 公開するのは `index.html` / `data/` / `assets/` の3つ（＋任意で`docs/`）。
> クローラ（`tools/`）は自宅PCに残す運用でOK（公開してもしなくても動作に影響なし）。

---

## どれを選ぶ？

| 方法 | 手軽さ | 更新 | プライバシー | 向いている人 |
|---|---|---|---|---|
| **A. Netlify Drop** | ◎ ドラッグ&ドロップ | 再ドロップ | 公開URL（推測されにくい） | とにかく早く出したい |
| **B. GitHub Pages** | ○ git操作 | `git push` | 公開URL（無料枠） | 更新を続けたい・URL固定 |
| **C. Cloudflare Pages ＋ Access** | △ 設定多め | git/直接 | **限定公開（メール認証）** | 自分だけに絞りたい |

まず出すなら **A**、長く運用するなら **B**、人に見られたくないなら **C** です。

---

## A. Netlify Drop（最速・gitなし）

1. スマホでもPCでも、ブラウザで **https://app.netlify.com/drop** を開く
2. 無料アカウントでサインイン（Google等でOK）
3. **`メダカ図鑑` フォルダをそのままドラッグ&ドロップ**
4. 数十秒で `https://xxxx.netlify.app` のURLが発行される → スマホ・他PCで開けばOK
5. **更新するとき**：データを更新したら、同じサイトの「Deploys」→ フォルダを再ドロップ（上書き公開）

※ URLは推測されにくいものの技術的には公開です。パスワードは有料。完全に隠したいなら方法C。

---

## B. GitHub Pages（更新しやすい・URL固定）

前提：GitHubアカウント（hiro5618-design）、PCにGit。
※ `git init`・初回コミット（ブランチ`main`）は**準備済み**です。残りは下記だけ。

1. GitHubで**空のリポジトリ**を作成
   - 右上「＋」→ New repository
   - Repository name：`medaka-zukan`
   - Public を選択
   - **「Add a README」「.gitignore」「license」は付けない**（空のまま。付けると push で衝突します）
   - Create repository
2. このフォルダのターミナル（Git Bash）で：
   ```
   cd "C:\Users\0300036\Documents\メダカ図鑑"
   git remote add origin https://github.com/hiro5618-design/medaka-zukan.git
   git push -u origin main
   ```
   ※ 初回 push でGitHubのログイン画面が出たらサインインしてください（ブラウザ認証）。
3. GitHubのリポジトリ → **Settings → Pages** → Source「Deploy from a branch」→ Branch を `main` / `(root)` にして Save
4. 数分後、`https://hiro5618-design.github.io/medaka-zukan/` で公開 → スマホ・他PCで開ける
5. **更新するとき**：`git add . && git commit -m "更新" && git push` だけ

※ 無料枠のPagesはURLが公開です。限定したい場合は方法C。

---

## C. Cloudflare Pages ＋ Access（自分だけに限定・無料）

1. Cloudflareアカウントを作成
2. Pages に方法Bのリポジトリを接続（または直接アップロード）
3. **Cloudflare Access** で「許可するメールアドレス（自分のGmail等）」を設定
   → そのメールで認証した人だけ閲覧可能（=実質プライベート）

やや設定は多いですが、**自分のスマホ・PCだけで見たい**場合はこれが確実です。

---

## 更新の反映について

- 図鑑データはクローラ（自宅PC・週1土曜）やパイプラインで更新されます
- 更新を公開サイトに反映するには **再デプロイ操作が必要**（A=再ドロップ／B・C=`git push`）
- 将来、この「更新→自動デプロイ」まで自動化することも可能（後日の拡張）
