# 相場収集の手順（自宅PC用）

メダカ図鑑の「相場（価格）」を楽天市場APIで集めるための作業手順です。
**自宅PC・自宅回線で行ってください。**（会社PCのIPを楽天に登録しないこと）

---

## 全体の流れ

```
① リポジトリを自宅PCに用意
② 楽天ウェブサービスでアプリを登録（グローバルIPが必要）
③ 認証情報を2ファイルに保存
④ 準備確認スクリプトで「要対応0件」にする
⑤ 相場を収集（分割実行でOK）
⑥ 公開に反映
```

---

## ① リポジトリを自宅PCに用意

初回のみ：

```
git clone https://github.com/hiro5618-design/medaka-zukan.git
```

2回目以降（最新にする）：

```
cd メダカ図鑑のフォルダ
git pull
```

> 認証情報のファイルは含まれません（各PCで作成します）。

---

## ② 楽天ウェブサービスでアプリを登録

まず**自宅のグローバルIP**を調べます。

```
python tools\price_setup_check.py
```

一番上に表示される `グローバルIP = ○○○.○○○.○○○.○○○` が登録する値です。

登録先： https://webservice.rakuten.co.jp/

フォームの入力内容は **`docs\楽天API登録メモ.txt`** にコピペ用でまとめてあります。
要点だけ再掲：

| 項目 | 入力する値 |
|---|---|
| アプリケーション名 | メダカ図鑑 |
| アプリケーションURL | https://hiro5618-design.github.io/medaka-zukan/ |
| アプリケーションタイプ | **API/バックエンドサービス** |
| 許可されたIPアドレス | 上で調べた**自宅の**グローバルIP |
| 予想QPS | 1 |
| APIアクセススコープ | **楽天市場APIのみ**チェック |

> 画像認証はご自身で入力してください（代行できません）。

---

## ③ 認証情報を2ファイルに保存

アプリ登録後、アプリ情報ページから2つの値をコピーします。

| 値 | 形式 | 保存先ファイル |
|---|---|---|
| アプリケーションID | UUID形式（`xxxxxxxx-xxxx-…`） | `tools\rakuten_app_id.txt` |
| アクセスキー | `pk_` で始まる文字列 | `tools\rakuten_access_key.txt` |

**ファイルには値だけを書いて保存**します（余計な文字や改行は不要）。

> どちらも `.gitignore` で除外済みです。GitHubには絶対に上がりません。
> アフィリエイトIDは相場収集には不要です。

---

## ④ 準備確認

```
python tools\price_setup_check.py
```

次の6点を自動で確認します。

1. グローバルIP
2. 認証ファイルの有無
3. 形式が正しいか
4. **認証ファイルがGitに漏れていないか**
5. 楽天APIへの疎通テスト（実際に1回だけ検索）
6. 対象品種数と所要時間

最後に「**要対応: 0件**」と出れば準備完了です。

### つまずいたときの対処

| 症状 | 原因と対処 |
|---|---|
| HTTP 400 / 401 / 403 | IP未登録・IP変更・キーの取り違え・スコープ未選択。表示されたIPを楽天側に登録し直す |
| アプリIDが20桁の数字 | 旧API用。旧APIは2026-05-14に停止済み。新しいアプリを作り直す |
| アクセスキーが `pk_` で始まらない | アプリIDと取り違えている可能性 |

## ⑤ 相場を収集する

全1241品種で約26分かかります。**分割して少しずつ**進めるのがおすすめです。

```
python tools\price_crawler.py --status done --limit 100
```

続きから再開：

```
python tools\price_crawler.py --status done --resume
```

残り全部：

```
python tools\price_crawler.py --resume
```

### 仕様（安心して止められます）

- **充実登録（done）の品種から先に**回るので、途中で止めても価値の高い品種から埋まります
- `Ctrl+C` で中断してもそこまでの結果は保存されます
- 50件ごとに自動保存されます
- `--resume` を付ければ、その日すでに取得した品種は自動で飛ばします
- 楽天のマナーに従い1.2秒に1リクエストに抑えています

### オプション一覧

| オプション | 意味 |
|---|---|
| `--status done` | 充実登録済みの780品種だけ（約16分） |
| `--status draft` | 調査中の461品種だけ |
| `--limit N` | 先頭N件だけ実行 |
| `--resume` | 本日取得済みを飛ばす |
| `m004 m003` | 品種IDを直接指定（テスト用） |

## ⑥ 公開に反映

```
deploy.bat
```

をダブルクリックするか、Claudeに「公開して」と伝えてください。

図鑑の品種詳細ページに「相場」として、卵／針子／成魚の
最安・平均・最高・件数・最安出品リンクが表示されます。

---

## 補足

- 集めるのは**楽天市場のみ**です。メルカリ・ヤフオクは規約上、自動収集しません
  （必要なら `docs\相場手入力メモの方法.md` の手順で補えます）
- ステージ判定（卵／針子／成魚）は出品タイトルからの推定なので、誤判定はあり得ます。
  件数と最安出品リンクを併記しているので、元の出品を確認できます
- 卵は「10個入り」など数量がバラバラです。価格は出品価格のまま載せ、
  数量のばらつきは備考欄に記載する仕様です
- 派生品種（例：白幹之ヒカリダルマメダカ）は出品が無く0件になることが多いです。
  これは異常ではありません
