## 3. データ設計（専門家①②の指摘を反映）

### 3-1. 読み込み方式 🔴
- データは **JavaScriptファイルに `window.MEDAKA_DATA = {…}` の形で埋め込む**
- `fetch` を使わず `<script>` タグで読むため、**ダブルクリックで確実に開く**（file://問題を回避）。Web公開時もそのまま動く

### 3-2. 品種ID 🟠
- 品種名ではなく **不変のID（`m001` 形式）** を主キーにする
- 品種名・読み・別名は「表示用の属性」として持つ（改名・表記ゆれに強い）

### 3-3. 2階建てデータ構造 🔴（図鑑用＋予測用）
品種データは「表現型」と「遺伝子型」を分けて持つ。

- **表現型タグ（図鑑・絞り込み用）**：見た目の分類（JMA第5版準拠）
- **遺伝子型メモ（交配予測用）**：ホモ/ヘテロ等。当面は空欄可。確実に分かる形質から埋める

### 3-4. 品種データ項目（1品種あたり）

| 区分 | 項目 | 備考 |
|---|---|---|
| 識別 | ID／品種名／読み／別名 | IDは不変 |
| 分類（表現型） | 体色／柄／虹色素胞／体型／ヒレ変化／目の変化 | JMA第5版の6カテゴリ |
| 参考タグ | 見本サイト系の形質タグ | 「参考」と明示、補助扱い |
| 系統 | 系統（血統）名／親品種IDリスト | 品種名と別管理（②の指摘） |
| 由来 | 作出者／作出年／作出の経緯 | 執筆係が数段落で |
| 説明 | 特徴の説明文 | 執筆係が数段落で |
| 飼育 | 難易度（★1〜5）／飼育のポイント | |
| 固定率 | 値＋出所＋調査時点 | 単一値にしない（②③の指摘） |
| 写真 | 表示方式（link/self）／外部リンクURL／自前ファイル名／出典／利用可否 | 二重化（①③の指摘） |
| 予測用 | 遺伝子型メモ | 第2フェーズ用・空欄可 |
| 記録枠 | 自分の飼育記録／交配実績 | 第2フェーズ用・空欄可 |
| 価格枠 | 価格履歴への参照 | 第2フェーズ用 |
| 出典 | 情報源URL・参照日 | 必須記録 |

## 6. サブエージェント全体像（コスト最適化済み）

| 係 | 役割 | モデル | 主なツール | 稼働 |
|---|---|---|---|---|
| ① 品種情報係 medaka-researcher | 由来・系統・特徴・飼育情報のWeb調査 | **Haiku** | Read, Grep, Glob, WebSearch, WebFetch | 第1〜 |
| ② 執筆係 medaka-writer | 解説文（説明＋作出の経緯を数段落）の執筆 | **Sonnet** | Read, Write, Edit | 第1〜 |
| ③ 図鑑作成係 zukan-builder | データ追記＋HTML反映（テンプレ流し込み） | **Haiku** | Read, Write, Edit, Glob | 第1〜 |
| ④ 校正係 zukan-checker | 表記ゆれ・記入漏れ・出典/利用可否の機械チェック | **Haiku** | Read, Grep, Glob | 第1〜 |
| ⑤ 写真管理係 photo-manager | 写真の出所・利用可否記録／穴埋めリスト／将来の整理・軽量化 | **Haiku** | Read, Write, Grep, Glob | 第1〜 |
| ⑥ 相場調査係 market-researcher | 楽天API＋公開検索で参考価格→履歴に追記 | **Haiku** | Read, Write, WebSearch, WebFetch | 第2〜（定義のみ今回） |

### コスト方針
- 単価の高いSonnetは**②執筆係のみ**。他は全部Haiku
- ツールを最小限に絞る／指示文を短く保つ／**一度調べたら再調査しない**（結果はデータに保存）／受け渡しを型化／コスト重視時は並列しすぎない
- CLAUDE.md等はプロンプトキャッシュを効かせるため頻繁に書き換えない

## 7. 交配予測（第2フェーズ・専門家②の指摘を反映）

- 予測は **「参考・目安」と明示**。計算対象は**遺伝の型が確実な形質だけ**（例：アルビノ・ヒカリ体型・ダルマ・透明鱗などの劣性形質）
- 複雑な形質（ラメ量・体外光・三色の柄など）は「複雑なため予測対象外」と正直に表示
- 進め方：A 形質早見表 → C 自分の交配実績記録 → B メンデル自動計算
- 3-3の「遺伝子型メモ」と3-4の「親品種IDリスト」がこの機能の土台

---

## 8. 相場・価格（第2フェーズ・専門家③の指摘を反映）

- **合法・安全優先**：楽天＝公式API（主軸／**「楽天市場」出典表記を自動付与**・レート制限順守）、その他＝公開検索の参考価格まで。メルカリ・ヤフオク等は**手入力の相場メモ欄**で補完（規約違反の自動収集はしない）
- **スナップショット記録型**：「ID・調査日・サイト・最安/平均/最高・件数・出典URL」を価格履歴に追記。回数を重ねて推移を育てる（過去に遡ったデータは取得不可）
- 定期実行での自動蓄積は図鑑完成後に検討

---

## 9. 写真・著作権方針（専門家③の指摘を反映）

- **外部リンク参照が基本**。自サイトに他人の画像を複製しない
- 見本サイト（改良メダカWEB図鑑＝めだかの館運営）は**無断転載・画像直リンク・インラインフレーム禁止**。**記事ページへの通常リンクはOK**なので、画像ではなく記事ページにリンクする
- **将来の切替枠**：データに「表示方式＝link/self」を持ち、自前撮影・許可取得写真が増えたらリンクを自前画像へ差し替えるだけで移行できる（リンク切れ対策も兼ねる）
- すべての写真・情報に**出典・参照日・利用可否**を必須記録
- 解説文は**自分の言葉で要約**が基本。原文引用は最小限＋出典明記（引用の要件順守）
