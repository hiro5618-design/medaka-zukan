■ 発行後の手順（2026-07-18改訂：楽天API刷新対応）
========================================================

※2026年2月に楽天ウェブサービスが刷新され、認証情報が
  「アプリケーションID＋アクセスキー」の2点セットになった。
  旧API（app.rakuten.co.jp、アプリID20桁数字）は2026-05-14停止済み。

1. アプリ情報ページで次の2つをコピー
     ・アプリケーションID … UUID形式（例 xxxxxxxx-xxxx-…）
     ・アクセスキー       … pk_ で始まる文字列（目のアイコンで表示）
   ※アフィリエイトID（ドット区切り）は相場収集には不要。

2. 次の2ファイルを作り、それぞれの値だけを書いて保存する
     メダカ図鑑\tools\rakuten_app_id.txt      ← アプリケーションID
     メダカ図鑑\tools\rakuten_access_key.txt  ← アクセスキー

   ※両ファイルとも .gitignore で除外済み。
     GitHub（公開リポジトリ）には絶対に上がらないので安全。
   ※環境変数 RAKUTEN_APP_ID / RAKUTEN_ACCESS_KEY に設定してもよい。

3. 相場を収集する
     cd "メダカ図鑑のフォルダ"
     python tools\price_crawler.py            ← 全46品種
     python tools\price_crawler.py m004       ← 品種ID指定（テスト用）

4. 結果は data\price-history.js に追記され、
   図鑑の詳細ページに「相場」として表示される
     ステージ：卵 / 針子 / 成魚
     項目：最安 / 平均 / 最高 / 件数 / 出所URL ＋ 備考

5. 公開反映（どちらでも）
     ・deploy.bat をダブルクリック
     ・またはClaudeに「公開して」と伝える

========================================================
■ 注意点
========================================================

・アプリIDは公開リポジトリに絶対にコミットしない（設定済み）
・グローバルIPが変わるとAPIが弾かれる。その時は同じフォームでIPを更新する
・ステージ判定は出品タイトルからの推定なので誤判定があり得る
  （件数と最安出品リンクを併記しているので元出品を確認できる）
・卵は「10個入り」など数量がバラバラ。価格は出品価格のまま、
  数量のばらつきは備考欄に記載する仕様
・楽天のみの相場。メルカリ・ヤフオクは規約上自動収集しない（手入力メモで補完）
