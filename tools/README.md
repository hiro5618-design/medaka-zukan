# 新品種 差分クローラ（Layer 1）

中核ソースを巡回して**新しい品種名だけ**を見つけ、`inbox`（未処理リスト）に貯めるツールです。
判断（裏取り・分類・図鑑への登録）は人＋Claudeが行う「Layer 2」で、ここは**発見の自動化**だけを担います。LLMを使わないのでほぼ無料で動きます。

## ファイル
| ファイル | 役割 |
|---|---|
| `medaka_crawler.py` | 本体。RSS取得→品種名抽出→既知と差分→inbox追記 |
| `sources.json` | 巡回する情報源リスト（中核3ソース） |
| `inbox.md` / `inbox.json` | 見つかった新品種候補（未処理） |
| `crawler-log.txt` | 実行ログ（日時・件数） |
| `register-task.bat` / `unregister-task.bat` | 週1自動実行の登録／解除 |

## 情報源（中核3ソース・すべてRSS）
- 改良メダカWEB図鑑 新着（`/feed/`）… タイトル「〇〇とは…」から品種名
- みんなのメダカ品種図鑑 新着（`/feed/`）… タイトルが品種名そのもの。ハウスネームに強い
- めだかの館 通販 新着（`?mode=rss`）… 商品名から品種名（在庫表記は除去）

## 使い方
```
python medaka_crawler.py
```
実行すると `inbox.md` に**新規候補だけ**が追記されます（既知の品種・既出候補は自動で除外）。

## 週1自動実行
`register-task.bat` をダブルクリック（または実行）すると、Windowsタスクスケジューラに
「毎週土曜 9:00 に実行」で登録されます。解除は `unregister-task.bat`。
（すでにこのタスクは登録済みです。曜日/時刻を変えたい時に使ってください）
※PCが起動していれば、Claudeを開いていなくても自動で走ります。

## 見つけた候補の処理（Layer 2）
1. `inbox.md` を開いて新規候補を確認
2. 図鑑に入れたいものは、Claudeに「inboxの〇〇を調査→登録して」と依頼
   → researcher/writer/builder/checker のパイプラインで裏取り・登録
3. 登録した候補は `inbox.json` の該当 `status` を `"done"` に変更（次回以降 `inbox.md` に出なくなる）

## 情報源を増やすには
`sources.json` の `sources` に追記します。
- `mode`：`toha`（〇〇とは）／`title`（タイトル＝品種名）／`colon`（商品名：在庫）
- `must_contain`：タイトルにこの語を含む項目だけ対象（空なら全件）

## マナー・注意
- 個人利用の低頻度・礼儀正しい巡回（User-Agent明記・アクセス間隔2秒）
- robots.txt / 各サイトの利用規約を尊重すること
- X・Instagram は自動対象外（規約・ログイン壁）。発見は人のタレコミで補完
- 画像は転載せず、参考リンク（記事・商品ページ）で扱う
